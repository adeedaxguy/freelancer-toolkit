## Quick answer

Before requesting indexing, confirm the page is allowed to index, self-canonical, internally linked, aligned with one search intent, and useful enough to deserve discovery. Indexing a weak page faster does not make it stronger.

## Decision table

| Audit stage | Checks | Why order matters |
| --- | --- | --- |
| Crawl basics | Robots, status code, canonical | Blocked pages cannot perform |
| SERP fit | Intent, format, answer depth | Wrong format rarely wins |
| Metadata | Title, description, H1 | Improves relevance and CTR |
| Content | Sections, examples, FAQs | Supports usefulness |
| Discovery | Internal links and sitemap | Helps crawlers find the page |

## Step-by-step workflow

1. Confirm HTTP 200 status and indexable robots settings.
2. Check canonical points to the intended URL.
3. Rewrite title and description around real search intent.
4. Add answer-first copy and scannable H2 sections.
5. Add internal links and only then request indexing if needed.

## Common mistakes to avoid

- Do not copy a competitor number without checking your own inputs.
- Do not treat a calculator result as final if the underlying assumption is wrong.
- Do not skip the follow-through: proposal, scope, audit fix, file check, or submission check.
- Do not optimize only for the cheapest or smallest number if quality, compliance, or margin matters.

## FAQ

### Should I request indexing before fixing on-page issues?

No. Fix the page first so the next crawl sees the strongest version.

### How many internal links should a page have?

Use enough relevant links for discovery and user flow. Quality matters more than count.

### Can this checklist be used for blog posts and tools?

Yes. It applies to any indexable page, including calculators, guides, and landing pages.
